## [1.0.0] — 2024-10-15

### Added

#### Core Platform
- Multi-tenant object storage platform with organization-scoped isolation
- Single-part file upload (up to 100 MB) via `PUT /api/v1/buckets/{id}/objects/{key}`
- Resumable multipart upload (up to 5 GB total, 500 MB per part) via `/api/v1/uploads/*`
- SHA-256 content-addressed deduplication with reference counting
- Object versioning — all PUT operations create a new `ObjectVersion`, previous versions are preserved
- Soft-delete with restore: objects are marked `is_deleted = true`, data is not immediately purged
- HTTP Range request support for partial downloads and resumable client downloads
- HMAC-SHA256 signed URLs with TTL, optional IP CIDR restriction, and download count limits
- Bucket management: create, list, delete

#### Authentication and Authorization
- RS256 asymmetric JWT — auth-service is the sole private-key holder
- JWKS endpoint at `/.well-known/jwks.json` for public key distribution
- Access token TTL: 15 minutes; refresh token TTL: 7 days
- Refresh token rotation with token family theft detection
- Multi-device logout (`POST /auth/logout-all`) revokes all refresh token families
- RBAC roles: `OWNER`, `ADMIN`, `EDITOR`, `VIEWER` scoped per organization
- Account lockout after 5 consecutive failed login attempts
- Redis-based JWT blacklist for immediate access token revocation on logout

#### Async Processing Pipeline
- Kafka-based event-driven processing triggered on every successful upload
- Parallel processor execution via Java 21 Virtual Thread executor
- Image thumbnail generation (Thumbnailator — JPEG/PNG/GIF/WebP)
- Video thumbnail generation (FFmpeg integration)
- PDF first-page preview rendering (Apache PDFBox)
- Virus scanning (EICAR pattern detection; ClamAV-ready interface)
- Metadata extraction (Apache Tika — EXIF, document properties)
- Dead Letter Topics for all processing events; failed events do not block the pipeline
- `file.uploaded` topic with 16 partitions for horizontal processing scale

#### Observability
- Micrometer metrics at `/actuator/prometheus` on all services
- Custom business metrics: upload throughput, processing pipeline latency, deduplication rate
- OpenTelemetry distributed tracing — trace context propagated across all service boundaries
- `X-Correlation-ID` header propagation (generated by NGINX if not provided by client)
- JSON-structured access logs on NGINX
- Pre-built Grafana dashboard: VaultFlow Platform Overview
- Prometheus scrape configuration for all services

#### Infrastructure
- Multi-service Docker Compose with infrastructure (PostgreSQL, Redis, Kafka, Zookeeper, Jaeger, Prometheus, Grafana)
- Kafka topics auto-created with appropriate partition counts on first startup
- PostgreSQL schema managed by Flyway versioned migrations
- Monthly-partitioned `audit_logs` table for scalable audit log retention
- NGINX API gateway with per-endpoint rate limiting zones, streaming I/O (no buffering), and security headers
- Kubernetes manifests: Deployment, Service, Ingress, HPA, PDB for auth, upload, and download services
- 7-stage GitHub Actions CI/CD pipeline: code quality → unit tests → integration tests → security scan → Docker build → staging deploy → production deploy
- Trivy and OWASP Dependency Check security scanning in CI

#### Developer Experience
- `install.sh` — automated dependency installer for Ubuntu/WSL
- `start.sh` — one-command local stack startup
- Detailed `docs/LOCAL_DEVELOPMENT.md` including JWT key sharing guidance
- `docs/RUNBOOK.md` with 6 incident playbooks and capacity planning queries
- `docs/ENGINEERING_DECISION_RECORDS.md` with 8 documented architecture decisions
- k6 load test script at `infrastructure/load-test.js`

#### Common Library
- Shared Kafka event schemas: `FileUploadedEvent`, `FileProcessedEvent`, `AuditEvent`
- Exception hierarchy with HTTP mapping: `ResourceNotFoundException`, `QuotaExceededException`, `InvalidUploadException`, `ConflictException`, `StorageException`
- `GlobalExceptionHandler` producing consistent `ApiErrorResponse` JSON
- `JwtTokenProvider` auto-configuration publishable as Spring Boot starter
- `ChecksumUtil`: SHA-256 computation and two-level content-addressed path routing
- `CorrelationIdFilter`: MDC propagation for structured logging
